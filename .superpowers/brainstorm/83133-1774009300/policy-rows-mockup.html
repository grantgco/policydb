Status Pill Colors — Including Custom Statuses

When users add custom statuses in Settings, they get auto-assigned colors from a rotating palette

Status Pill Color System

Built-in statuses (fixed colors):

Not Started In Progress Quoted Pending Bind Bound

Custom statuses added by user (auto-colored from palette):

Declined Under Review Awaiting Docs Not Renewing Referred

Color assignment logic: Built-in statuses have fixed color mappings. Custom statuses rotate through a palette of 8 Tailwind color families (pink, sky, yellow, rose, teal, indigo, orange, lime) based on their index in the config list. No manual color picking needed.

Contenteditable Cell Behavior

Cell Interactions
Click cell Blue bottom-border appears. Cell becomes editable. Cursor placed.
Type + blur PATCH saves. If server reformats (e.g., "$45000" → "$45,000"), cell flashes green briefly.
Tab Moves to next editable cell in the row. Shift+Tab goes back.
Empty cell Shows placeholder text via CSS ::before.
Status pill click One click saves. Selected pill fills with color. No dropdown, no form.
Date field Native <input type="date"> styled as borderless inline. Change event saves.
