Programs — How should they appear?

A program is a single aggregate record (is_program=1 on policies). Here's how it would look on the client detail page alongside regular policies.

Option A — Programs as a distinct section (Recommended)

Programs get their own collapsible section above the project/location policy groups. Visually distinct — signals "this is an umbrella over the whole account." Each program shows aggregate metrics + participating carriers.

Option B — Programs mixed into policy groups

Programs appear as rows in the "Corporate / Standalone" group alongside regular policies, distinguished by a badge. Simpler but programs can get lost among individual policies.

Reconciliation behavior

  • Matching: Fuzzy match works same as any policy — client name + policy type (e.g., "D&O")
  • Multiple matches: Several statement lines from different carriers can map to the same program record
  • Partial reconciliation: Track matched premium vs total — show "3 of 4 carriers reconciled, $140K of $180K matched"
  • Import: When a statement line matches a program, tag it as a program match (no special import changes needed)

New fields on policies table

  • is_program INTEGER DEFAULT 0 — flag, like is_opportunity
  • program_carriers TEXT — comma-separated carrier list (e.g., "AIG, Chubb, Zurich")
  • program_carrier_count INTEGER — number of participating carriers
