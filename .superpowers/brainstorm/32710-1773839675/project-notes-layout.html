How should the project notes expand?

The collapsed state always shows a truncated rendered preview. Pick how the full note opens.

Option A — Inline expand (Recommended)

Clicking "Show more" grows the note panel in place. An EasyMDE editor appears when you click Edit. Auto-saves on change. No page navigation required.

▶ Main Street Condos 3 policies Edit · Log · Email

Executive Summary — 12-unit luxury condo tower, completed 2022. Primary risk concerns include high-value finishes (~$800K per unit) and a...

↕ Expands here — EasyMDE editor, auto-saves, "▲ Collapse" button below

Option B — Modal / overlay

Clicking "Show note" opens a full-screen modal with the rendered note and an Edit toggle. Good for very long documents, keeps the client page layout stable.

▶ Main Street Condos 3 policies Edit · Log · Email

Executive Summary — 12-unit luxury condo tower...

Opens as a centered overlay covering the page

Option C — Dedicated project page

The project name becomes a link to a dedicated /clients/{id}/projects/{name} page with full editor, activity, and policy list. More powerful but adds navigation.

▶ Main Street Condos ↗ 3 policies

Executive Summary — 12-unit luxury condo tower...

Click name → full project page with editor, activity, policy list

My suggestions (regardless of layout choice)

  • Auto-save — paste in LLM output, it saves automatically (800ms debounce, same as policy scratchpad). No save button needed.
  • Markdown rendering in preview — the collapsed 3-line preview renders the markdown (bold, bullets) so it's readable at a glance.
  • Character/word count — small indicator like "1,240 words" in the header when a note exists, signals the note has substance.
  • Timestamp — "Updated Mar 18 at 2:14pm" shows freshness of the LLM summary.
